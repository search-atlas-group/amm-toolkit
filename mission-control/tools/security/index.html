Isolation

The Sandbox

Runs suspicious code in a sealed zone. The code executes — but can't call home and can't touch your real files.

macOS sandbox-exec
Built into every Mac since 10.5. No Docker, no installs. The mac-sandbox.sb profile blocks all network and restricts writes to /tmp.
Docker container (optional)
Full OS-level isolation. --network none, --read-only, --memory 512m, --cap-drop ALL. Container is destroyed after the test.
What gets captured
All network connection attempts (even if blocked), file writes outside /tmp, processes spawned, and permission denial errors.
When it triggers
Automatically on MEDIUM+ risk (score ≥ 31), or anytime via the --sandbox flag. Container is destroyed — nothing persists.

Progressive Enhancement

The 4 Scan Tiers

The scanner never fails — click each tier to see what it can detect.

Tier 0 · Always available
AI Analysis Only
Requires: Just Claude Code — nothing else to install
  • Surface recon via GitHub API (age, stars, size)
  • Install hook inspection (postinstall, prepare)
  • Reverse shell pattern detection
  • Obfuscated code detection (eval/base64 chains)
  • Crypto miner signature scanning
  • Credential theft pattern scanning (.ssh, clipboard)
  • Embedded binary detection
  • Committed secret files (.env, .pem, .key)
What you'll see when this tier runs

A short summary: repo age, star count, size, whether install hooks exist, and which static patterns matched. Findings are quoted at file and line. Verdict lands in 5–15 seconds.

Best for · a first look at any unfamiliar repo before cloning.
Tier 1 · Basic tools
Pattern Scanning
Requires: git + curl + jq (usually pre-installed)
  • Everything in Tier 0
  • Clone to isolated /tmp directory
  • Full codebase grep across all files
  • Dependency file deep inspection
  • CI workflow file analysis (.github/)
  • File type scanning for hidden executables
What you'll see when this tier runs

Tier 0 output, plus a list of dependency anomalies and any CI workflow steps that fetch or execute external code. Hidden binaries outside expected asset folders are listed by path.

Best for · checking repos you intend to clone and read locally.
Tier 2 · Recommended
Full Automated Scan
Requires: trivy · gitleaks · trufflehog · semgrep  (bash Scripts/install-security-tools.sh)
  • Everything in Tiers 0 & 1
  • CVE database lookup (Trivy)
  • Secrets in current code (Gitleaks)
  • Secrets in full git history (TruffleHog)
  • Professional SAST rules (Semgrep)
  • npm audit / pip-audit / bundle audit
What you'll see when this tier runs

A verdict plus per-tool counts: how many secrets Gitleaks found in the working tree, how many TruffleHog found in history, how many CVEs Trivy mapped, and which SAST rules Semgrep tripped. Each finding lists severity, file, and detail.

Best for · the default choice when you'll actually install or run the repo.
Tier 3 · Maximum coverage
Sandbox Execution
Requires: Docker Desktop  or  built-in macOS sandbox-exec (no install)
  • Everything in Tiers 0, 1 & 2
  • Install runs in a sealed container
  • All network connections blocked
  • Network attempts captured and logged
  • File system writes monitored
  • Container destroyed after test
What you'll see when this tier runs

Tier 2 output, plus a behavioral report: did install try to reach the network, did it try to write outside /tmp, did it spawn unexpected processes. Any blocked attempts are listed with the destination it was trying to reach.

Best for · packages with install hooks, or any repo with an unclear postinstall step.
