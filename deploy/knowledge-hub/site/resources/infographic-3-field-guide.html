📄 What Each File Does
CLAUDE.md

Your agency's permanent instructions. Claude reads it every session. Persona, clients, tools, rules.

AI_RULES.md

Hard limits: never publish without approval, never expose keys, always confirm before destructive actions.

PRD.md

Product/Project Requirements Doc. What are the agency's goals this quarter? How do we measure success?

PLAN.md

Current sprint or 30-day plan. What's being worked on, by whom, and by when.

README.md — Your North Star

One page that explains everything: what this folder is, how to start a session, which commands to use, where things are saved, and who to ask if something breaks. If a new team member opened this folder cold, README.md should get them running in 10 minutes.

🔒 Data Loss Prevention
Add .env to .gitignore before anything else
Always save reports with date stamps: 2026-03-acme.md
Use /export before closing any important session
Never let Claude publish content before you see it first
Never store API keys in any file other than .env

📋 Dos & Don'ts
✓ Always Do
Name every session before working
Check /mcp at session start
Use @filename to reference documents
/clear between every client
Save reports with date stamps
Read Claude's plan before approving
Use /compact in long sessions
Export outputs before closing
✕ Never Do
Mix clients in the same session
Write vague prompts ("fix my SEO")
Paste API keys into the chat
Approve without reading Claude's plan
Skip /clear and wonder why reports are wrong
Store secrets outside .env
Correct Claude 3x — clear & rewrite instead
Ignore MCP connection errors
◈ The Golden Rule of Prompting

Strong: "Update Q3 SEO report for Acme in /reports/acme/2026-03.md — add top 10 keyword rankings from Search Atlas, format as executive summary with MoM comparison, confirm before saving."
💡 Cool Tips
Power Moves for Agency Owners
Each tip saves you 5–30 minutes per week
🖼
Paste screenshots directly Drop a screenshot of any dashboard, design, or error into Claude. It reads images — no need to describe what you see.
📎
Reference files with @ Type @brief.pdf or @keywords.txt to pull any file into context. Works with PDFs, docs, spreadsheets.
🔄
Resume cross-day sessions Named sessions survive shutdowns. /resume brings back full context — no re-explaining your project.
🧠
Plan mode before big tasks Type /plan or press Shift+Tab. Claude thinks first, shows you the approach — you approve before anything happens.
🤫
Quick side questions with /btw Ask something without adding it to the task history: /btw what does "canonical URL" mean?
⚡
Warp Drive for repeat tasks Save your most-used claude commands to Warp Drive. One click to launch your monthly reporting workflow.
🔧 Back on Track — Troubleshooting
Feeling lost? Breathe. One of these will fix it.
Claude is giving wrong or confusing answers
Use /clear and start fresh with a cleaner, more specific prompt. You haven't lost anything — just context.
A tool (Search Atlas, ClickUp) isn't working
Type /mcp to see connection status. If it's red, remove and re-add: claude mcp remove [name] then re-add it. Takes 30 seconds.
Session is slow or Claude seems "foggy"
Use /compact to summarize and free memory. If still bad, export outputs, /clear, and resume fresh with a summary.
I can't find a report Claude generated
Ask: "Where did you save the last report?" Or check /export history. Going forward: always end sessions by saving to /reports/.
