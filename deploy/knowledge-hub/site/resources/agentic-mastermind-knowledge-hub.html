File Structure

Repository Layout & Token Hygiene

Know what each file does — and which ones Claude Code should never read. A clean structure = fast, cheap sessions.

Manual Setup

If the wizard fails

If setup.sh doesn't work or you're adding the MCP server manually, edit this file directly. The header name is case-sensitive — X-API-KEY exactly.

~/.claude/settings.json
{ "mcpServers": { "searchatlas": { "type": "url", "url": "https://mcp.searchatlas.com/sse", "headers": { "X-API-KEY": "YOUR_API_KEY_HERE" } } } }
.env (optional integrations)
# Required SA_API_KEY=sk-your-key-here # Slack (optional) SLACK_WEBHOOK_URL=https://hooks.slack.com/... SLACK_WEBHOOK_SEO=https://hooks.slack.com/... # Discord (optional) DISCORD_WEBHOOK_URL=https://discord.com/api/... # Email via Resend (optional) RESEND_API_KEY=re_your_key EMAIL_FROM=you@yourdomain.com # Circle (optional) CIRCLE_API_KEY=your_key CIRCLE_COMMUNITY_ID=12345
⚠️ Key Rotation Gotcha: Editing .env alone does NOT update the running MCP config. After changing your API key, re-run bash setup.sh OR manually update ~/.claude/settings.json. That file is what Claude Code reads at runtime.

Skill files are markdown documents in your skills/ folder. Each one teaches Claude exactly how to run a repeatable workflow. Invoke with use skill [name] for [client].

Applies client-specific tone, vocabulary, and style to all content. Pulls from brand vault data. Prevents generic "plumbing page" output.

CRO Audit
Conversion rate optimization audit for a specific page — reviews CTAs, structure, intent match, and friction points. Outputs prioritized fixes.

Monthly reporting structure — KPI definitions, executive summary format. Pulls ranking movement, traffic changes, writes commentary at client's reading level.

Error Reference

Troubleshooting Guide

Find your error message below. Each card walks you through the exact fix — no guessing.

T1
Authentication Error
"credentials not provided" or "authentication failed"
1
Run the verify script first: bash scripts/verify-setup.sh
2
Check that ~/.claude/settings.json has a searchatlas block with X-API-KEY (all caps, hyphen — not underscore)
3
Confirm key is valid: log in at searchatlas.com → Settings → API Keys
4
Test the raw connection:
curl -H "X-API-KEY: YOUR_KEY" https://mcp.searchatlas.com/sse
⚠️ Key Rotation: If the key was rotated, re-run bash setup.sh with the new key. Editing .env alone is not enough — it must propagate to settings.json.
T2
Tool Not Found
"Tool not found" or tool returns wrong data type
1
Run schema discovery on any tool to see current operations:
op: "help", params: {}
2
If project_management returns content projects instead of OTTO projects → retry with otto_project_management prefix
3
Check deprecated tool names — common renames: OTTO_Project_Management → project_management, BrandVaultTools → brand_vault
T3
Parameter Validation Error
"Parameter validation error" / "missing required field: [field]"
1
Read the error message carefully — it contains the correct schema with exact field names and types
2
Common gotchas by tool group:
brand_vault (CRUD) → brand_vault_uuid (NOT vault_id) brand_vault (read-only) → hostname (NOT brand_vault_uuid) holistic_audit → domain (plain string) gbp_locations_crud → location_id (integer, NOT uuid) organic / backlinks → project_identifier (domain, URL, or ID) project_management → project_id (UUID)
3
Call the tool with empty params to get live schema: op: "list_otto_projects", params: {}
T4
404 / Resource Not Found
"404", "not found", empty results for a known resource
1
Run /my-account to get a fresh list of all current IDs — never use hardcoded IDs
2
Verify the resource exists in the SearchAtlas dashboard
3
If recently created: wait 1–2 minutes and retry (propagation lag is normal)
T6
Connection Timeout / Hangs
Claude Code hangs, SSE drops, no response from MCP
1
Test raw connectivity:
curl https://mcp.searchatlas.com/sse
2
Restart Claude Code — it will re-establish the SSE connection
3
On corporate/office networks: SSE on port 443 may need to be explicitly allowed by IT. Test on a different network first.
T7
Command Not Found
Typing /run-seo shows nothing or "command not found"
1
Verify commands are installed:
ls ~/.claude/commands/ # Should show 13 .md files
2
If missing: re-run bash setup.sh from the repo directory
3
After a repo update: git pull origin main then bash setup.sh again
T8
Workflow Step Stuck / Async Timeout
Workflow stops at a step for too long / polls forever
1
Check last status from the polling call: PENDING/IN_PROGRESS = still running (wait), SUCCESS = process result, FAILED = retry that step only
2
Normal wait times: content generation 30–90s per phase, PPC keyword clustering 2–5 min
3
If PENDING for over 15 minutes: the task likely failed silently. Retry only the failed step — do NOT restart the whole workflow
T10
Notification Not Sending
/send-slack, /send-discord, /send-email fails silently
1
Verify env vars are set in .env
2
Test Slack webhook directly:
curl -X POST -H 'Content-type: application/json' \ --data '{"text":"test"}' YOUR_SLACK_WEBHOOK_URL
3
Named Slack channels: env var must be SLACK_WEBHOOK_CHANNELNAME in ALL CAPS
4
For Resend email: confirm domain is verified OR use onboarding@resend.dev for testing. Check spam folder first.
5
For Circle: CIRCLE_COMMUNITY_ID must be the numeric ID — not the slug or name
Session Issues

When Claude Feels Off

These aren't API errors — they're session state issues. Different fixes than tool errors.

Claude giving wrong or confusing answers
Use /clear and start fresh with a cleaner, more specific prompt. You haven't lost anything — just stale context. If you've corrected Claude twice on the same thing, always clear and rewrite instead of correcting a third time.
Session is slow or Claude seems "foggy"
Run /compact to summarize and free memory. If still bad: /export your outputs, then /clear and resume with a fresh summary. Run /compact proactively at 65–70% context bar.
Can't find a report Claude generated
Ask: "Where did you save the last report?" or check /export history. Going forward: always end sessions with explicit save instructions like "save to /reports/[client]/[month].md".
Everything feels overwhelming
Start with 3 things: 1) open Warp, 2) type claude, 3) say what you need in plain English. Everything else is optional until you're ready. You don't need to understand the code — just describe the outcome clearly.
Common Pitfalls

6 Mistakes That Break Everything

These aren't beginner errors — experienced users hit all of them. Know the fix before you hit the problem.

P1
Cold-start syndrome
Every session opens with zero memory. Without AI_RULES.md and CLAUDE.md in place, you re-explain your world every time and results drift.
Fix: build AI_RULES.md first
P2
The silent misroute
Claude says "done" but the file saved to the wrong folder — no error, no warning. Happens when opening Claude Code without confirming working directory.
Fix: ask "what is cwd?"
P3
Only using one level of CLAUDE.md
Root-only: every client gets the same generic output. Client-only: no agency baseline, standards reset every session. You need both — always.
Fix: root + per-client
P4
Generic prompts produce generic output
"Write a page about plumbing services" sounds like every other plumbing page. Less specific input = Claude falls back to default patterns. Skill files and CLAUDE.md close this gap.
Fix: use skill files
P5
Building site pages outside Search Atlas
Building outside Website Studio loses version control, OTTO integration, one-click publish, and workflow templates. You trade 10 min setup for hours of manual work downstream.
Fix: use Website Studio
P6
Treating skill files as one-time setup
Skill files are living documents. If keyword research improves, the skill should improve. If a client changes brand voice, update the file. Stale skills produce stale output.
Fix: review on process change
