Why This Matters

The Problem This Solves

When someone says "install this library" or "clone this repo," they're asking you to run code written by someone you've never met. Most of the time that's fine. But occasionally, that code is designed to:

  • Steal your passwords or API keys stored on your computer
  • Mine cryptocurrency silently in the background (your electricity, their profit)
  • Create a backdoor that lets a stranger control your computer remotely
  • Send your files to someone else's server

This tool checks for all of that before you run anything. You don't need to understand what it's checking — Claude will tell you what it found in plain English.

How To Use It

Run This in Claude Code

Open Claude Code, type this, and press Enter:

/security-scan https://github.com/owner/repo-name
Replace the URL with the repo you want to check.

Claude will do the rest and give you a plain-English answer.

Also works with
/security-scan owner/repo  —  short form
/security-scan /path/to/local/folder  —  for repos already on your machine

Flags You Can Add

  • --quick — Skip the sandbox, static checks only. Done under 2 minutes.
  • --sandbox — Force sandbox even if the scan looks clean.
  • --share — Send the report to your default Slack channel when done.
  • --no-install — Skip the tool install offer; work with what's already available.

How It Works

The Scan Runs in Layers

From fastest to most thorough — each layer adds coverage:

1
Before touching any code
Claude checks the repo's public profile: How old is it? How many people use it? A brand-new repo with zero users claiming to be a popular tool is an immediate red flag.
2
Reading the code
Claude reads the code like a security analyst would — looking for code that connects to outside servers during install, reads your SSH keys or passwords, is intentionally scrambled to look harmless, or hides programs inside what should be text files.
3
Automated security tools (optional, free)
Checks against a database of known vulnerabilities, scans the entire commit history for leaked passwords, and runs professional-grade code analysis. Claude will offer to install these tools in about 2 minutes if you don't have them.
4
Sandbox test (optional, no extra install)
Runs the code inside a completely sealed-off zone on your Mac — no internet access, can't touch your real files. Claude watches what the code tries to do and reports anything unusual. Your Mac has this built in; no Docker required.
FAQ

Common Questions

"I'm not technical. Can I still use this?"
Yes. Claude explains everything in plain English. You don't need to know what "SAST" or "CVE" means — Claude will translate.
"How long does it take?"
AI-only: 1–2 minutes  •  With tools: 3–5 minutes  •  With sandbox: 5–8 minutes
"Does it slow down my computer?"
No. The scan runs on code that hasn't been installed yet — nothing is active on your machine while it runs.
"Can I scan a repo I already cloned?"
Yes. Run /security-scan /full/path/to/the/folder instead of a URL.
"What if the repo is private?"
The surface reputation check won't work, but Layers 2–4 still work once you have a local copy. Pass the folder path instead of a URL.
"Should I scan every repo I use?"
Scan anything from a source you don't know personally, haven't used before, or that someone sent you unexpectedly. You don't need to scan major open-source projects like React or Django — but do scan any small utility, plugin, or tool shared in a Discord, Slack, or email.
"What if Claude says it can't check something?"
That's honest — it means a specific check wasn't possible. The verdict will reflect what was actually checked, and Claude will tell you what it couldn't verify.

Emergency

If You Find Something Dangerous

  1. Don't panic. Finding it before running it means it didn't do anything yet.
  2. Don't install it. Close the terminal window / folder.
  3. Report it using the "Report repository" button on GitHub (flag icon on the repo page).
  4. Tell your team — if this was shared in a work channel, others might have already installed it.
  5. If you already ran it: change any API keys or passwords you've used in the last 24 hours. Claude will give you a specific list of what to revoke.
