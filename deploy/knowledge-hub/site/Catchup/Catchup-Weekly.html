File Structure Is the Root Cause
Matt Bailey · Justin Day · Bryan Fikes · JD (screen share)
Setup File Structure Workflows
Context: Homework from Manick — build satellite sites and ensure your main site is bulletproof. This session diagnosed the #1 root cause of issues across the group.
Member Updates
Bryan Fikes
Switched to building sites as raw HTML/CSS via Claude Code. Zero hiccups. Now his approach for all multi-page builds.
Justin Day
Website Studio working well for single location landing pages — 3 prompts, done.
Key Session Demo

JD’s File Structure Deep-Dive (screen share)

  • Your root directory (the folder you created at setup) is sacred. Every project lives inside it.
  • For new client projects: create a subfolder → open Claude → start working. For quick tasks: stay in root.
  • Always check your folder path. File structure is the most common root cause of push failures — not Claude, not the MCP.
  • SA is building vault-level architecture directly into the platform.

Post-Session Learning Loop (Justin Day)

  • After every work session, have Claude review what happened: what went wrong, what improved, whether file integrity is intact.
  • Claude adds insights to memory and gets better at your workflow over time.

Other Key Points

  • Ask Claude to “list your MCP workflows” — SA has pre-built playbooks you can trigger directly.
  • Google Drive integration: segment your Drive the same way you segment image galleries. MCP reads and uploads directly.

Patterns

Seven things that keep coming up.
Read these before your first session.

01
File Structure = Root Cause
Almost every platform push failure traces back to a wrong directory. Check your folder path before anything else.
02
Copy Before Design
Write all content before design begins. Unlocks parallel work and dramatically reduces client revision cycles.
03
Google Drive Integration
Give SA tools MCP access to your Google Drive. Segment it the same way you segment your galleries or client deliverables — the platform reads and uploads directly.
04
Satellite Site Strategy
Manick’s ongoing homework: build review-style directories and third-party authority sites that link to your primary. Depth-1 and depth-2 pages set the domain’s quality score.
05
Post-Session Learning Loop
After every work session, have Claude review the session log, flag what went wrong, and add insights to memory. Your AI gets better at your workflow over time.
06
Role-Frame Claude Explicitly
“You are a [domain expert] providing a [output type] on behalf of [client].” The role context shapes format, tone, and depth of output.
07
SA MCP Playbooks
Ask Claude to “list your MCP workflows.” SA has pre-built playbooks you can trigger directly from Claude Code without building from scratch.
